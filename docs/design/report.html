🏗️ 架构设计要点

  • 微服务架构：按DDD领域驱动设计拆分：用户服务、商品服务、订单服务、营销服务、支付服务、消息服务、社区服务
  • API网关：Spring Cloud Gateway统一入口，限流、鉴权、路由、日志
  • 服务治理：Nacos注册发现 + Sentinel熔断降级 + SkyWalking链路追踪
  • 数据策略：MySQL主从读写分离 + Redis缓存 + ES搜索引擎 + Canal数据同步
  • 异步处理：RabbitMQ处理订单异步、消息通知、库存扣减等耗时操作
  • 容器化部署：Docker + K8s编排，支持自动扩缩容和滚动更新

🎯 核心功能（MVP）

对标小象超市的履约链路：

  • 消费者端：浏览商品、下单支付、配送时间轴、签收码确认
  • 履约端：扫码入库、按库位分拣、配送员抢单、取货送达
  • 管理后台：商品/库存/订单/配送员审核、履约监控
  • 前置仓：常温/冷藏/冷冻分区，赏味期与库位溯源
  • 开放配送：任何人申请成为配送员，审核后可抢单
  • 本期不做：团长分销、积分商城、地图追踪、多仓（二期）

📅 开发路线图

Phase 1: MVP 履约闭环（单仓单小区）

预计 6-8 周

  • 项目初始化与基础架构搭建
  • 邻选·购：下单支付 + 配送追踪页
  • 邻选·履约：入库 + 分拣 + 抢单配送
  • 管理后台：商品/订单/配送员审核
  • 后端 API + 订单状态机
  • 山屿西山著内测验收

Phase 2: 效率与体验

预计 4-6 周

  • 波次分拣、地图位置
  • 订阅消息全链路推送
  • 配送费结算、数据报表
  • 优惠券 / 秒杀

Phase 3: 规模扩展

预计 6-8 周

  • 多前置仓、跨小区
  • 智能拣货路径
  • 积分 / 分销（可选）
  • 后端微服务拆分
