📐 架构设计理念

采用微服务 + 前后端分离架构，遵循以下核心原则：

🏛️ 领域驱动设计 (DDD)

按业务领域划分微服务边界：用户、商品、订单、营销、支付等，每个服务独立部署、独立扩展。

⚡ 高性能设计

Redis缓存热点数据、CDN加速静态资源、ES搜索引擎、RabbitMQ异步削峰，保障秒杀等高并发场景。

🔒 安全第一

API网关统一鉴权、JWT令牌管理、SQL防注入、XSS防护、敏感数据脱敏加密、HTTPS全链路加密。

📈 可观测性

SkyWalking链路追踪、Prometheus指标采集、Grafana可视化监控、ELK日志聚合，全链路可观测。

☁️ 部署方案说明

🏢 推荐云平台

方案A：阿里云 — ECS + RDS + Redis + OSS，成熟的电商云基础设施
方案B：腾讯云 — 与微信生态集成更紧密，小程序云开发
方案C：混合部署 — 核心服务自建 + CDN/OSS上云

📦 容器化部署

所有服务Docker容器化，K8s编排管理。支持：
• HPA自动扩缩容（应对秒杀峰值）
• 滚动更新（零停机发布）
• 健康检查与自动重启
• 资源限制与隔离

🔄 CI/CD 流程

代码提交 → GitHub Actions 自动构建 → 单元测试 → SonarQube扫描 → Docker镜像构建 → 推送镜像仓库 → K8s滚动更新

📈 容量规划 (MVP)

• 应用服务器: 2C4G × 3台
• MySQL: 4C8G RDS
• Redis: 4G 集群版
• ES: 2C4G × 3节点
• 带宽: 5Mbps (可弹性扩展)

🔄 核心业务流程补充

⚡ 秒杀流程优化

Redis预扣库存 → Lua脚本原子操作 → 异步下单 → MQ削峰 → 最终一致性。避免直接操作MySQL造成瓶颈。

👥 团长分销流程

用户通过团长分享链接进入 → 绑定上下级关系 → 下单后自动计算佣金 → 订单完成结算 → 可提现到微信零钱。

🔄 退款流程

用户申请退款 → 团长/平台审核 → 调用微信退款接口 → 恢复库存 → 退回优惠券 → 更新订单状态。

📦 拼团流程

团长开团 → 分享链接 → 团员参团 → 人数达标自动成团 → 批量生成订单 → 统一发货。超时未成团自动退款。
