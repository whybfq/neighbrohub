## 阶段 1 · 云服务器与域名（必做）

### 1.1 购买服务器

| 项 | 建议 |
|----|------|
| **云厂商** | 阿里云 / 腾讯云 / 华为云（选离北京用户近的华北区域） |
| **规格（起步）** | 2 核 4G、40G SSD、3–5M 带宽（日单 <500 可够用） |
| **系统** | Ubuntu 22.04 LTS 或 CentOS Stream |
| **预估费用** | 约 ¥80–200/月（新用户活动价更低） |

### 1.2 域名与备案

- [ ] 购买域名（如 `linshe.com` / `neighbrohub.cn`）
- [ ] **ICP 备案**（国内服务器上架小程序 API **必须**，约 7–20 工作日）
- [ ] 公安备案（部分省份要求，备案通过后按当地要求办理）

### 1.3 HTTPS 与反向代理

- [ ] 申请免费 SSL（Let's Encrypt 或云厂商证书）
- [ ] Nginx / Caddy 反向代理：`https://api.你的域名.com` → `localhost:8090`
- [ ] 管理后台静态站：`https://admin.你的域名.com` → `admin-web/dist`
- [ ] 对象存储（可选）：腾讯云 COS / 阿里云 OSS 存商品图、Banner

### 1.4 数据库（替换内存存储）

- [ ] 购买云数据库 MySQL 8.0（或与 API 同机自建，生产建议 RDS）
- [ ] 设计表：用户、商品、订单、库存、分拣任务、配送任务、积分、地址
- [ ] 迁移脚本 + 备份策略（每日自动备份）

## 阶段 2 · 微信小程序（必做）

### 2.1 注册与配置

| 端 | 说明 |
|----|------|
| **消费者小程序** | 独立 AppID，类目：社区团购 / 生鲜电商等 |
| **履约端小程序** | 独立 AppID，类目：工具 / 企业内部（或同类目） |

- [ ] 微信公众平台注册两个小程序（企业主体，需营业执照）
- [ ] 微信认证（¥300/年/小程序，或同主体第二个半价）
- [ ] 配置 **request 合法域名**：`https://api.你的域名.com`
- [ ] 配置 uploadFile / downloadFile 域名（若上传图片）
- [ ] 修改各端 `project.config.json` 中的 AppID
- [ ] 生产 API：见各端 `.env.example` / `config/prod.js` 中的 `TARO_APP_API`

### 2.2 微信登录

- [ ] 消费者：`Taro.login` → 后端用 `code` 换 `openid` / `session_key`
- [ ] 履约员：独立账号体系或企业微信 / 手机号 + 验证码
- [ ] 手机号：接入 `getPhoneNumber` 组件 + 后端解密（需认证小程序）

### 2.3 微信支付（必做才能真收钱）

- [ ] 开通 **微信商户号**（需营业执照、对公账户）
- [ ] 商户号绑定两个小程序 AppID
- [ ] 后端：统一下单 API + 支付结果 **回调验签** + 幂等处理
- [ ] 前端：`Taro.requestPayment` 调起支付（替换当前「确认支付」弹窗模拟）
- [ ] 退款接口（售后需要）

### 2.4 订阅消息

- [ ] 申请模板：支付成功 / 备货中 / 配送中 / 已送达
- [ ] 用户下单、支付后引导授权订阅
- [ ] 订单状态变更时调用 `subscribeMessage.send`

### 2.5 提审材料

- [ ] 隐私政策、用户协议页面（小程序内可打开）
- [ ] 个人信息收集说明
- [ ] 商品真实图片（替换 Emoji 占位）
- [ ] 食品/生鲜类目资质（若涉及）
- [ ] 体验版内部全流程走通后再提审

## 阶段 6 · 部署架构（推荐）

```
                    ┌─────────────────┐
  用户微信 ────────►│ 消费者小程序      │
  作业员微信 ──────►│ 履约端小程序      │
  运营浏览器 ──────►│ admin-web (Nginx) │
                    └────────┬────────┘
                             │ HTTPS
                    ┌────────▼────────┐
                    │ Nginx + SSL     │
                    │ api.xxx.com     │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ Node/API 服务   │
                    │ (PM2 / Docker)  │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
       ┌──────▼──────┐ ┌─────▼─────┐ ┌──────▼──────┐
       │ MySQL       │ │ Redis(可选)│ │ OSS 图片    │
       └─────────────┘ └───────────┘ └─────────────┘
```

### 部署命令参考

```bash
# 服务器上
git clone https://github.com/whybfq/neighbrohub.git
cd neighbrohub/server && cp .env.example .env  # 编辑后
npm ci && npm run build && pm2 start npm --name neighbrohub-api -- start

cd ../admin-web && npm ci && npm run build
# Nginx root 指向 admin-web/dist

# 小程序：本地 config/prod.js 改 API 后
cd mini-program && npm run build:weapp
# 微信开发者工具上传代码 → 提审
```

## 优先级排序

```
1. 买服务器 + 域名备案 + HTTPS
2. 数据库 + 持久化 API
3. 两个小程序 AppID + 合法域名
4. 微信登录 + 微信支付
5. 真图 + 提审
6. 订阅消息 + 监控 + 客服
```

完成阶段 1–3 即可小范围试运营；阶段 4–6 可并行准备。
